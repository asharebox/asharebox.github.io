建立自己的“开源大脑” repo git for AI

为了解决在使用 AI 时反复输入背景信息、数据分散在不同平台的低效问题，构建了一个基于本地 Git 仓库的纯文本知识库——AIStudio。

核心观点与做法：

  1. Git 作为协作协议：放弃封闭的云端笔记软件，改用 Git 仓库管理知识。因为现在的 AI 具备读取整个工作区的能力，通过 Git 仓库，AI 能直接获取完整上下文，甚至通过 Agent 自主提交产出。

  2. 工程化目录结构：建立了一套标准化的目录（如 Inbox、ContentFactory、System、Product 等），分别管理灵感、内容生产流水线、SOP 与提示词库、产品开发等，且目录结构可随需求自然生长。

  3. “大脑库”与“外挂系统”分离：为了避免 Git 仓库臃肿，仓库内只保留信息密度高的文档（架构、思路、SOP）；而体积大的代码项目或多媒体素材则存放在外部独立仓库，仅在主仓库中保留链接和说明文档。主仓库作为指挥中心，外挂系统作为执行单元。

通过这种方式建立了一个可迁移、永久保存的“外脑”，实现了人与 AI 的高效协作。

Step by Step Guide (实操指南)

建立自己的“开源大脑”，请按以下步骤操作：

第一步：搭建基础设施

  1. 创建 Git 仓库：在本地新建一个 Git 仓库（命名为 AIStudio 或 MyBrain）。

  2. 放弃炫酷笔记软件：决定将核心知识沉淀转移到本地纯文本文件中，确保 AI 可以直接读取。

第二步：构建目录骨架

在仓库根目录下创建以下基础文件夹（可根据个人需求微调）：

  • Inbox / Thinking：存放碎片化灵感和日常深度思考。

  • ContentFactory：建立内容生产线。

    • 子目录建议：Ideas（选题库）、Pipeline（发布流程）、Scripts（脚本）。

  • System（核心大脑皮层）：

    • 存放 SOP（标准作业程序）。

    • 存放工作流模板。

    • 重点：创建 PromptHub 目录，专门存放给 AI 看的设定和提示词。

  • Product：产品化内容跟踪与专题输出。

  • Automation：存放自动化脚本和工作流日志。

  • Admin：管理账号信息、工具列表、环境配置等。

第三步：迁移与填充内容

  1. 输入背景信息：将你以前需要反复告诉 AI 的信息（“我是谁”、“项目规范”、“写作风格”）整理成 Markdown 文件，放入 System 目录。

  2. 整理现有资产：将散落在各处的笔记、文档迁移到对应目录。不需要追求完美分类，先放进去，后续让 AI 帮忙整理。

第四步：实施“中枢+外挂”策略

处理大文件或独立项目时，遵循以下原则：

  1. 主仓库（中枢）：只放文档。例如：架构设计图、需求文档、README、复盘笔记。

  2. 外部项目（外挂）：对于大型代码库、视频素材等，单独建立仓库或存储。

  3. 建立链接：在主仓库的相关目录中，通过文档或 README 链接到这些外部仓库。确保 AI 读取主仓库时，能看到“地图”，知道去哪里找重资产。

第五步：建立协作习惯

  1. 日常 Commit：每当有新想法、跑通新流程或项目有更新时，务必 Commit 到仓库。这不仅是备份，更是为了让 AI 获取最新上下文。

  2. 唤醒 AI：在本地 IDE 或支持读取工作区的 AI 工具中打开该目录。

  3. 指令交互：告诉 AI：“读取当前工作区，根据 System 里的规范执行……” 你不再需要每次重复输入背景信息。

第六步：迭代优化

  • 不要试图一次性设计完美结构，让目录随着使用自然生长。

  • 定期让 AI 根据现有内容整理目录结构或优化 SOP。
