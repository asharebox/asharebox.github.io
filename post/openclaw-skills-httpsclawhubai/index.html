最适合新手先装的 20 个 OpenClaw Skills

三个标准：

  • 上手门槛低

  • 使用场景高频

  • 能明显提升第一次体验

1. tavily-search：先给 Agent 接上“外脑”

ClawHub 地址：https://clawhub.ai/arun-8687/tavily-search

如果一个 Agent 不能查最新信息，它的能力天花板会立刻降下来。

所以，对大多数人来说，第一优先级不是“让它更聪明”，而是先让它看到外部世界。

tavily-search 这类联网搜索 Skill 的价值非常直接：

  • 查最新资料

  • 验证事实

  • 找官网文档

  • 补充上下文

尤其你拿它做调研、写作、选型、找案例时，联网能力基本就是基础设施。

一句话建议：如果你只装一个技能，先装 tavily-search。

2. ontology ：让它别聊完就忘

这里其实有两个选择，一个ontology，一个agent-memory

ClawHub 地址：

  • ontology 可参考：https://clawhub.ai/maverick-software/surrealdb-knowledge-graph-memory

  • agent-memory 可参考：https://clawhub.ai/kgnvsk/agent-memory-store

很多人第一次用 Agent 都会有一个共同吐槽：

“怎么我刚说过的东西，它转头就忘？”

这就是为什么记忆类 Skills 很重要。它解决的不是“回答更漂亮”，而是“协作更连续”。

如果你更看重结构化知识沉淀，可以优先看 ontology；如果你更在意对话连续性和长期上下文，agent-memory 会更直接。

这类能力特别适合：

  • 长周期项目

  • 个人知识管理

  • 连续多轮协作

  • 固定风格写作与输出

一句话建议：想把 Agent 从“工具”变成“助手”，尽早补上 ontology 或 agent-memory。

3. self-improving-agent：让它越用越顺手

ClawHub 地址：https://clawhub.ai/pskoett/self-improving-agent

有些 Skill 的价值，不在于第一次有多惊艳，而在于你连续用几次之后，会明显感觉：它在逐渐“会做事”了。

self-improving-agent 往往会帮助 Agent：

  • 记录失败经验

  • 调整执行方式

  • 优化下一轮动作

  • 降低重复犯错概率

对于新手来说，这类技能的最大意义不是“炫技”，而是让你更快体会到 Agent 工作流的核心魅力：不是一次回答，而是持续协作。

4. proactive-agent：从“等你发话”到“自己往前推”

ClawHub 地址：https://clawhub.ai/halthelobster/proactive-agent

大多数新手刚开始用 Agent 时，本质上还是在“问答模式”。你问一句，它答一句；你再催一步，它再走一步。

而 proactive-agent 这类 Skill，真正改变的是交互关系。

它会让 Agent 更像一个会自己拆任务、自己推进、自己检查的执行者，而不是一个只会等待指令的聊天窗口。

适合的场景包括：

  • 做调研方案

  • 规划任务步骤

  • 拆解复杂目标

  • 持续推进多阶段工作

一句话建议：想感受“Agent 味儿”到底在哪里，尽早试试 proactive-agent。

5. summarize：信息摄入效率直接翻倍

ClawHub 地址：https://clawhub.ai/steipete/summarize

如果你平时经常处理这些东西：

  • 长文章

  • PDF

  • 视频

  • 图片材料

  • 网页链接

  • 文档说明

那 summarize 会非常高频。

它的价值并不是简单“帮你缩短字数”，而是帮你更快完成三件事：抓重点、提结构、形成下一步动作。

很多时候，真正浪费时间的不是“没找到信息”，而是“找到以后来不及消化”。而 summarize，正好是把“阅读”变成“提炼”的加速器。

ClawHub 地址：https://clawhub.ai/steipete/github

一句话建议：开发者新手包里，github 基本属于“早装早省事”。

# porteden email

Use `porteden email` (alias: `porteden mail`) for email management across multiple accounts. **Use `-jc` flags** for AI-optimized output.

If `porteden` is not installed: `brew install porteden/tap/porteden` (or `go install github.com/porteden/cli/cmd/porteden@latest`).

Setup (once)

- **Browser login (recommended):** `porteden auth login` — opens browser, credentials stored in system keyring
- **Direct token:** `porteden auth login --token <key>` — stored in system keyring
- **Verify:** `porteden auth status`
- If `PE_API_KEY` is set in the environment, the CLI uses it automatically (no login needed).

Common commands

- List emails (or --today, --yesterday, --week, --days N): `porteden email messages -jc`
- Filter emails: `porteden email messages --from sender@example.com -jc` (also: --to, --subject, --label, --unread, --has-attachment)
- Search emails: `porteden email messages -q "keyword" --today -jc`
- Custom date range: `porteden email messages --after 2026-02-01 --before 2026-02-07 -jc`
- All emails (auto-pagination): `porteden email messages --week --all -jc`
- Get single email: `porteden email message <emailId> -jc`
- Get thread: `porteden email thread <threadId> -jc`
- Send email: `porteden email send --to user@example.com --subject "Hi" --body "Hello"` (also: --cc, --bcc, --body-file, --body-type text, --importance high)
- Send with named recipient: `porteden email send --to "John Doe <john@example.com>" --subject "Hi" --body "Hello"`
- Reply: `porteden email reply <emailId> --body "Thanks"` (add `--reply-all` for reply all)
- Forward: `porteden email forward <emailId> --to colleague@example.com` (optional `--body "FYI"`, --cc)
- Modify email: `porteden email modify <emailId> --mark-read` (also: --mark-unread, --add-labels IMPORTANT, --remove-labels INBOX)
- Delete email: `porteden email delete <emailId>`

Notes

- Credentials persist in the system keyring after login. No repeated auth needed.
- Set `PE_PROFILE=work` to avoid repeating `--profile`.
- `-jc` is shorthand for `--json --compact`: strips attachment details, truncates body previews, limits labels, reduces tokens.
- Use `--all` to auto-fetch all pages; check `hasMore` and `nextPageToken` in JSON output.
- Email IDs are provider-prefixed (e.g., `google:abc123`, `m365:xyz789`). Pass them as-is.
- `--include-body` on `messages` fetches full body (default: preview only). Single `message` includes body by default.
- `--body` and `--body-file` are mutually exclusive. Use `--body-type text` for plain text (default: html).
- Confirm before sending, replying, forwarding, or deleting emails.
- Environment variables: `PE_API_KEY`, `PE_PROFILE`, `PE_TIMEZONE`, `PE_FORMAT`, `PE_COLOR`, `PE_VERBOSE`.

新手安装建议顺序

很多人一上来喜欢“一键全装”。坦白说，这通常不是最优解。

对新手最友好的顺序，应该是：

第一步：先补基础能力

优先装：

  • skill-vetter

  • tavily-search

  • weather

  • fetch

这一步的目标不是变强，而是先让系统可用、可验证、可继续。

第二步：再补执行能力

接着装：

  • summarize

  • proactive-agent

  • self-improving-agent

  • filesystem

  • pdf

这一步开始，你会明显感受到 Agent 从“会回答”变成“会做事”。

第三步：最后补场景能力

再根据你是谁，来选分支：

如果你偏开发：

  • github

  • ontology 或 agent-memory

  • python

  • shell

  • browser

如果你偏办公与内容：

  • gog

  • ontology 或 agent-memory

  • notion

  • calendar

  • email

如果你还在探索：

  • find-skills

  • slack

新手最容易踩的 4 个坑

1. 装得太多

技能不是越多越好。过多能力叠加，往往会带来：

  • 决策混乱

  • 调用变慢

  • 成本变高

  • 调试困难

新手最好的状态不是“全都有”，而是“先有 3 到 5 个最常用的”。

2. 一开始就追求高级玩法

自动化编排、复杂代理链、深度多工具协同，这些当然很酷。但如果你连基础工作流都还没跑顺，太早上复杂玩法，只会增加挫败感。

新手阶段最重要的不是“堆复杂度”，而是尽快建立稳定使用习惯。

3. 忽视权限与安全

任何能帮你做事的 Skill，本质上都意味着它可能拥有某种权限。

所以你至少要养成三个习惯：

  • 安装前看来源

  • 使用前看权限

  • 定期清理与更新

别因为图快，把长期风险一起带进来。

4. 没有围绕真实任务来装

最容易浪费时间的方式，就是围绕“别人说好用”来装技能。

更有效的方式是反过来问自己：

  • 我每天最重复的动作是什么？

  • 我最痛的低效环节是什么？

  • 我现在最需要被替代的，是查资料、做总结、写文档，还是项目协作？

技能应该服务任务，而不是让任务去迁就技能。

#!/usr/bin/env node

function usage() {
  console.error(`Usage: search.mjs "query" [-n 5] [--deep] [--topic general|news] [--days 7]`);
  process.exit(2);
}

const args = process.argv.slice(2);
if (args.length === 0 || args[0] === "-h" || args[0] === "--help") usage();

const query = args[0];
let n = 5;
let searchDepth = "basic";
let topic = "general";
let days = null;

for (let i = 1; i < args.length; i++) {
  const a = args[i];
  if (a === "-n") {
    n = Number.parseInt(args[i + 1] ?? "5", 10);
    i++;
    continue;
  }
  if (a === "--deep") {
    searchDepth = "advanced";
    continue;
  }
  if (a === "--topic") {
    topic = args[i + 1] ?? "general";
    i++;
    continue;
  }
  if (a === "--days") {
    days = Number.parseInt(args[i + 1] ?? "7", 10);
    i++;
    continue;
  }
  console.error(`Unknown arg: ${a}`);
  usage();
}

const apiKey = (process.env.TAVILY_API_KEY ?? "").trim();
if (!apiKey) {
  console.error("Missing TAVILY_API_KEY");
  process.exit(1);
}

const body = {
  api_key: apiKey,
  query: query,
  search_depth: searchDepth,
  topic: topic,
  max_results: Math.max(1, Math.min(n, 20)),
  include_answer: true,
  include_raw_content: false,
};

if (topic === "news" && days) {
  body.days = days;
}

const resp = await fetch("https://api.tavily.com/search", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify(body),
});

if (!resp.ok) {
  const text = await resp.text().catch(() => "");
  throw new Error(`Tavily Search failed (${resp.status}): ${text}`);
}

const data = await resp.json();

// Print AI-generated answer if available
if (data.answer) {
  console.log("## Answer\n");
  console.log(data.answer);
  console.log("\n---\n");
}

// Print results
const results = (data.results ?? []).slice(0, n);
console.log("## Sources\n");

for (const r of results) {
  const title = String(r?.title ?? "").trim();
  const url = String(r?.url ?? "").trim();
  const content = String(r?.content ?? "").trim();
  const score = r?.score ? ` (relevance: ${(r.score * 100).toFixed(0)}%)` : "";
  
  if (!title || !url) continue;
  console.log(`- **${title}**${score}`);
  console.log(`  ${url}`);
  if (content) {
    console.log(`  ${content.slice(0, 300)}${content.length > 300 ? "..." : ""}`);
  }
  console.log();
}

#!/usr/bin/env node

function usage() {
  console.error(`Usage: extract.mjs "url1" ["url2" ...]`);
  process.exit(2);
}

const args = process.argv.slice(2);
if (args.length === 0 || args[0] === "-h" || args[0] === "--help") usage();

const urls = args.filter(a => !a.startsWith("-"));

if (urls.length === 0) {
  console.error("No URLs provided");
  usage();
}

const apiKey = (process.env.TAVILY_API_KEY ?? "").trim();
if (!apiKey) {
  console.error("Missing TAVILY_API_KEY");
  process.exit(1);
}

const resp = await fetch("https://api.tavily.com/extract", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    api_key: apiKey,
    urls: urls,
  }),
});

if (!resp.ok) {
  const text = await resp.text().catch(() => "");
  throw new Error(`Tavily Extract failed (${resp.status}): ${text}`);
}

const data = await resp.json();

const results = data.results ?? [];
const failed = data.failed_results ?? [];

for (const r of results) {
  const url = String(r?.url ?? "").trim();
  const content = String(r?.raw_content ?? "").trim();
  
  console.log(`# ${url}\n`);
  console.log(content || "(no content extracted)");
  console.log("\n---\n");
}

if (failed.length > 0) {
  console.log("## Failed URLs\n");
  for (const f of failed) {
    console.log(`- ${f.url}: ${f.error}`);
  }
}

# Tavily Search

AI-optimized web search using Tavily API. Designed for AI agents - returns clean, relevant content.

## Search

```bash
node {baseDir}/scripts/search.mjs "query"
node {baseDir}/scripts/search.mjs "query" -n 10
node {baseDir}/scripts/search.mjs "query" --deep
node {baseDir}/scripts/search.mjs "query" --topic news
```

## Options

- `-n <count>`: Number of results (default: 5, max: 20)
- `--deep`: Use advanced search for deeper research (slower, more comprehensive)
- `--topic <topic>`: Search topic - `general` (default) or `news`
- `--days <n>`: For news topic, limit to last n days

## Extract content from URL

```bash
node {baseDir}/scripts/extract.mjs "https://example.com/article"
```

Notes:
- Needs `TAVILY_API_KEY` from https://tavily.com
- Tavily is optimized for AI - returns clean, relevant snippets
- Use `--deep` for complex research questions
- Use `--topic news` for current events
